WHY DATASETS FIRST

先学 `datasets`，是为了避免微调时盲目炼丹

微调效果差，经常不是模型小、显卡弱，而是样本字段、切分、tokenization、labels 或模板错了。 `datasets` 是你进入训练前的控制台。

你真正要学的是数据如何变成 loss

原始样本不会直接进入模型。它要经过字段选择、格式拼接、tokenizer 编码、padding/truncation、 labels 对齐。最后，只有 labels 中没有被 mask 的位置才会影响训练。

`map()` 是微调数据管线的主干

`map()` 把原始列变成训练列。分类任务里它负责 tokenize 文本；SFT 里它负责拼 prompt、 构造 answer labels；大数据场景里它还负责缓存、批处理和可复现。

本地电脑要先学“小而完整”

不要第一天就训练 7B。先用几十条样本、本地小模型，把 `load_dataset -> train_test_split -> map -> Trainer` 跑通。能让 20 条样本过拟合，说明管线大概率是通的。

源码不用先啃，先掌握 API 心智模型

第一阶段不需要读完 `arrow_dataset.py`。你要先知道 Dataset 是表、DatasetDict 是 split 容器、 Features 是 schema、IterableDataset 是流式数据入口。

OBJECT MODEL

四个对象先记住：Dataset、DatasetDict、Features、IterableDataset

把它们想成训练前的数据仓库结构。你不是在处理一堆 Python list，而是在操作一张可缓存、可切分、可映射的表。

Dataset 单个 split 的表
dataset[0]
取一行样本，通常是 dict。用于抽查原始字段和处理后字段。
dataset["text"]
取一列。适合快速检查文本、label 分布或长度。
dataset.map(fn)
生成新 Dataset。注意很多处理方法不是原地修改。
DatasetDict 多个 split 的容器
raw["train"]
训练集。训练参数更新来自这里。
raw["validation"]
验证集。不要和训练集重复，否则指标失真。
train_test_split
没有验证集时自己切分。学习阶段固定 seed。
Features 字段 schema 和类型系统
Value("string")
文本字段。SFT 的 instruction/input/output 常见类型。
ClassLabel
分类标签。可保留类别名和 id 的映射。
cast
把字段转换成训练期需要的类型。
IterableDataset 流式读取大数据
streaming=True
不完整下载到磁盘，边读边处理。适合大数据。
shuffle(buffer_size)
流式数据用近似 shuffle，不是普通随机访问。
DataLoader
可直接交给 PyTorch，但学习初期先用普通 Dataset。

from datasets import load_dataset from transformers import AutoTokenizer raw = load_dataset("json", data_files="data/sft.jsonl") raw = raw["train"].train_test_split(test_size=0.1, seed=42) tokenizer = AutoTokenizer.from_pretrained("sshleifer/tiny-gpt2") tokenizer.pad_token = tokenizer.eos_token def tokenize_batch(batch): texts = [x + "\n" + y for x, y in zip(batch["instruction"], batch["output"])] return tokenizer(texts, padding="max_length", truncation=True, max_length=256) tokenized = raw.map(tokenize_batch, batched=True, remove_columns=raw["train"].column_names)

CONCEPT DICTIONARY

这节课必须吃透的概念

这些词不是 API 名词表，而是你以后排查微调问题时会反复使用的判断工具。

seed

随机数种子。`seed=42` 的作用是让 split、shuffle 这类随机操作可复现。 同一份数据、同一个 seed，切出来的 train/validation 应该一致，方便比较实验。

test_size

验证集比例。`test_size=0.2` 表示 20% 样本进入验证集。本课 5 条样本会切成 4 条 train、1 条 validation。

DatasetDict

多个 Dataset 的字典容器，常见键是 `train`、`validation`、`test`。 训练代码通常要求你明确传入 train 和 eval split。

batched=True

`map` 一次处理一批样本，而不是一条样本。tokenizer 通常批处理更快，也更接近真实训练数据处理方式。

remove_columns

删除原始文本列，只保留训练需要的列。避免 Trainer 收到 `instruction/output` 这类模型不认识的字段。

max_length

固定序列最大长度。超过会截断，不足会 padding。SFT 中如果 max_length 太小，answer 会被截掉，模型学不到完整回答。

input_ids

模型真正看到的 token id 序列。它来自 prompt 和 answer 拼接后的编码结果。

attention_mask

标记哪些位置是真实 token，哪些位置是 padding。真实 token 是 1，padding 是 0。

labels = -100

PyTorch 交叉熵里常用的忽略标记。SFT 中把 prompt 部分设成 `-100`，表示这部分不参与 loss。

TASK FORMS

三类微调数据，不要混成一团

分类、生成、SFT 的数据看起来都像文本，但训练目标不同。目标不同，labels 构造就不同。

输入是文本，目标是类别

分类微调通常用 encoder 模型，例如 BERT。数据里要有文本列和 label 列，模型输出 logits， loss 来自分类头，不需要自己构造每个 token 的 labels。

{ "text": "这家店的服务很好", "label": 1 }
输入也是目标，下一个 token 预测

causal LM 微调里，模型读前面的 token 预测下一个 token。`labels` 通常和 `input_ids` 同长度，只是在你不想训练的位置设置 `-100`。

{ "prompt": "写一句关于春天的诗", "completion": "春风入旧巷，花影落新茶。" }
训练模型按指令回答

SFT 的关键不是把三列拼起来，而是只让回答部分贡献 loss。这样模型学习“看完指令后如何回答”， 而不是平均地背模板和用户问题。

{ "instruction": "解释什么是 LoRA", "input": "", "output": "LoRA 是一种参数高效微调方法..." }
SFT LABELS

SFT 最关键的问题：哪些 token 参与 loss

如果你只记住一个微调细节，就记住这个：`input_ids` 是模型看到的，`labels` 是模型被要求学会的。 `-100` 位置不参与 loss。

训练文本
### Instruction: 解释什么是梯度累积 ### Response: 梯度累积是在显存不足时，把多个 mini-batch 的梯度累加后再更新参数的方法。
labels 心智模型
Prompt 区域
labels = -100
Answer 区域
labels = token_id
检查方法
decode labels 中非 `-100` 的部分，应该只看到 assistant 回答。
def build_sft_features(example): prompt = f"### Instruction:\n{example['instruction']}\n\n### Response:\n" answer = example["output"] + tokenizer.eos_token prompt_ids = tokenizer(prompt, add_special_tokens=False).input_ids answer_ids = tokenizer(answer, add_special_tokens=False).input_ids input_ids = prompt_ids + answer_ids labels = [-100] * len(prompt_ids) + answer_ids return { "input_ids": input_ids[:256], "labels": labels[:256], "attention_mask": [1] * min(len(input_ids), 256), }

FAILURE MODES

数据管线常见失败模式

出现这些问题时，先回到 `datasets` 层排查，不要先改模型。

labels 全是 `-100`

模型没有任何回答 token 可学。检查 prompt/answer 边界。

训练和推理模板不一致

训练用 `### Response`，推理却用 chat template，会让输出不稳定。

验证集泄漏

eval loss 很好看，但真实泛化没有提升。检查重复样本。
