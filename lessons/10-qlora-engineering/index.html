LESSON 10 · QLORA ENGINEERING

QLoRA
不是 Mac 上的魔法开关

本课用本地脚本做训练工程规划：检测硬件、选择课程模型策略、读取安全的 Hugging Face 元数据，并估算 LoRA/QLoRA/DeepSpeed 方案的内存边界。

先划清 执行边界

QLoRA

4-bit NF4 冻结 base 权重，加 LoRA adapter 训练小增量。

CUDA + bitsandbytes

标准 HF QLoRA 路径依赖 CUDA 方向的 4-bit kernel。

Mac / MPS

可以学习 LoRA 和做规划，但不要声称 bitsandbytes QLoRA 原生可跑。

DeepSpeed

多 GPU 状态切分边界，不是 CUDA kernel 的替代品。

预算拆成 四块

Base weights

fp16 LoRA 约 2 bytes/param；NF4 double quant 规划可按约 0.56 bytes/param 初筛。

→
Adapter train state

LoRA A/B 权重、梯度、AdamW 两个 fp32 moments。通常远小于 base。

→
Activations

由 microbatch、seq length、hidden、layers 决定；checkpointing 用重算换显存。

peak ~= base_weights + lora_weights + lora_grads + optimizer_states + activations + safety_margin

训练工程 旋钮

microbatch

直接影响激活峰值。OOM 时先降它。

accum

提高有效 batch，不等价于提高单步峰值。

checkpoint

降低激活保存量，但 backward 更慢。

DeepSpeed 什么时候进场

场景优先选择注意
7B 单卡 QLoRACUDA + bitsandbytes + PEFT先 one-step smoke test。
13B/长上下文紧张降 seq/batch/rank，开 checkpointing先别把分布式复杂度拉进来。
多卡全参或大 LoRADeepSpeed ZeRO / FSDP确认量化、PEFT、保存格式兼容。
