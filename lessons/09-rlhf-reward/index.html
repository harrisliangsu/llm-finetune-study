流程

01加载偏好对
02加载 HF tokenizer/model
03训练 reward model
04计算 reference 分布
05初始化 policy
06PPO + KL 更新
07输出评估报告

关键概念

  • Reward model: 把同一 prompt 下的偏好对压缩成回答分数；本课用 prompt + response 规则特征做教学模型。
  • Reference model: 冻结基线，提供 KL 约束的锚点。
  • Policy: 当前被优化的回答分布。
  • KL: 控制 policy 不要为了刷 reward 过度偏离 reference。
  • PPO: 用 clipped ratio 控制每次策略更新幅度；本课固定更新前候选分布作为 pi_old。

看什么结果

  • 打开 report.md 看 PPO 前后指标。
  • 打开 policy_before.jsonl 和 policy_after.jsonl 看候选概率变化。
  • 打开 outputs/trace.json 看每一步 trace event。
