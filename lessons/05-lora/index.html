LESSON 05 · LORA ADAPTER

LoRA
冻结 base，只训练低秩 adapter

这一课不下载模型。我们继续使用本地 tiny causal LM，在 `lm_head` 上插入 LoRA A/B 矩阵，训练、保存并重新加载 adapter。

MODEL DOWNLOAD

这节课不需要下载模型

为什么不下载

学习 LoRA 的第一目标是看清机制：冻结 base 参数、只训练 adapter、保存 adapter、重新加载 adapter。tiny model 足够演示这些动作。

什么时候要下载

当你进入真实 LLM adapter 训练时，需要下载 base model，例如小型 Hugging Face causal LM。那会是下一课或后续扩展。

这节课的核心不是大模型框架

目标是用手写 tiny LoRA 看清 adapter 机制：base 权重冻结，adapter 作为可训练旁路提供 delta。

forward = base + delta

logits = lm_head(x) + lora_B(lora_A(x)) * alpha / r

和 Lesson 06 的边界

Lesson 05 不是真 Qwen / PEFT 训练，不讲工程化 adapter 注入；它只服务于理解 LoRA 为什么能“只保存增量、加载到 fresh base”。

判断实现是否正确：训练前后 base 参数应保持不变，checkpoint 里应只包含 LoRA A/B 相关权重。

GLOSSARY

专有词解释 + 简短例子

LoRA

作用：一种参数高效微调方法，不直接改完整模型权重，而是在目标线性层旁边加低秩分支来学习增量。

例子：本课不更新 lm_head.weight，只训练 lora_A/lora_B，前向变成 lm_head(x) + LoRA_delta(x)。

rank / r

作用：决定低秩矩阵的中间维度。r 越大，adapter 参数越多，表达能力越强；r 越小，越省显存和存储。

例子：本课 r=4，lora_A 把 hidden size 96 压到 4，lora_B 再映射回 vocab size 256。

alpha / scaling

作用：控制 LoRA 分支输出加到 base 输出上时的力度，常见缩放是 alpha / r。

例子：本课 alpha=8、r=4，所以 scaling 是 2，LoRA delta 会乘以 2。

adapter

作用：挂在 base model 上的小型可训练模块，负责学习新任务调整，base model 保留原有语言能力。

例子：本课 adapter 是 lm_head 旁边的 lora_A/lora_B，只有 1,408 个可训练参数。

frozen base

作用：base model 参数被冻结，不参与 optimizer 更新，用来降低训练成本并避免整体改坏原模型。

例子：TinyCausalLM 原始参数设为 requires_grad=False，训练时只更新 LoRA adapter。

delta

作用：LoRA 学到的权重变化量或输出修正量，不替换原始权重，只叠加到原输出上。

例子：xW^T + b + (xA^T B^T) * scaling 中最后一项就是 adapter 产生的 delta。

state dict

作用：PyTorch 保存参数的字典，key 是参数名，value 是张量；保存和加载 adapter 本质上就是读写这些张量。

例子：adapter checkpoint 里有类似 lora_A.weight、lora_B.weight 的参数。

adapter-only checkpoint

作用：只保存 adapter 参数，不保存 base model；体积小，但加载时必须配合同结构 base model。

例子：lora_adapter.pt 需要先创建 fresh TinyCausalLM，再加载 adapter。
