CONCEPT DICTIONARY

专有词解释 + 简短例子

这些词会在 tokenizer、SFT 数据构造和 loss 排查里反复出现。先记住它们的作用，再看代码会清楚很多。

tokenizer

把文本转成模型能处理的 token id，也能把 id decode 回文本。模型不直接读中文字符串。

例子
tokenizer("解释什么是梯度累积") 会输出 input_ids。
token

模型处理文本的最小片段，可能是一个字、词、子词、标点或特殊符号。

例子
"梯度累积" 不一定等于 4 个 token，真实切法由 tokenizer 决定。
vocab

词表，保存 token 和整数 id 的映射。vocab size 会影响模型输入嵌入表大小。

例子
本课本地 tokenizer 的 vocab size = 256。
input_ids

token 对应的整数序列，是模型真正看到的输入，会进入 embedding 层。

例子
"### Response:" 会被编码成一串整数 id。
attention_mask

标记真实 token 和 padding。真实 token 是 1，补齐出来的位置是 0。

例子
补齐到 96 长度时，后 25 个 padding 的 mask 是 0。
SFT labels

告诉 loss 哪些 token 要学习。prompt 通常设为 -100，answer 保留 token id。

例子
本课 prompt 13 个 token，所以 labels 前 13 个都是 -100。
prompt / response

prompt 是给模型看的指令和上下文，response 是希望模型学会生成的回答。

例子
### Instruction... 是 prompt；梯度累积是在... 是 response。
padding / truncation

padding 把短序列补齐，truncation 把超长序列截断，二者让 batch 形状稳定。

例子
max_length=96 时，不足会补 pad，超过会截断。
EOS

end-of-sequence，表示回答结束。SFT 中加 EOS 可以教模型在合适位置停止。

例子
回答末尾拼上 <eos>，decode labels 时应能看到它。
