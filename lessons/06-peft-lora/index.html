GLOSSARY

专有词解释 + 简短例子

PEFT

作用：Parameter-Efficient Fine-Tuning，参数高效微调工具链。它提供统一接口来做 LoRA、Prefix Tuning 等方法；LoRA 是 PEFT 支持的一种具体方法。

例子：本课用 LoraConfig 和 get_peft_model 把 Qwen 改造成只训练 LoRA adapter 的模型。

PEFT 方法地图

作用：PEFT 是一组省参数微调方法，不是单一算法。当前课程主线是 LoRA，后续可扩展到 AdaLoRA、IA3、Prompt/Prefix/P-Tuning。

例子：LoRA 训练 A/B 低秩矩阵；Prompt Tuning 训练 soft prompt；IA3 训练少量缩放向量。

Hugging Face model id

作用：定位 Hub 上模型的名字，通常是 组织或用户/模型名，用于下载 tokenizer、config 和权重。

例子：Qwen/Qwen2.5-0.5B-Instruct 是本课 model id，from_pretrained 会根据它找模型文件。

AutoModelForCausalLM

作用：根据模型配置自动选择 causal LM 类，适合加载“输入文本，预测下一个 token”的生成式语言模型。

例子：AutoModelForCausalLM.from_pretrained(model_id) 加载完整 Qwen base model，再交给 PEFT 注入 LoRA。

AutoTokenizer

作用：根据 model id 加载对应 tokenizer，把中文、英文和符号切成模型能理解的 token id。

例子：“介绍一下 LoRA”会被编码成一串整数 id，再喂给 Qwen 训练。

target_modules

作用：告诉 PEFT 要把 LoRA adapter 挂到哪些模块上。选得越多，可训练参数越多；选得太少，可调整空间可能不足。

例子：本课设置 target_modules=["q_proj", "v_proj"]，只改 attention 的 query/value 投影层。

q_proj / v_proj

作用：Transformer attention 里的线性投影层。调整它们会影响模型“看哪里”和“取出什么信息”。

例子：LoRA 在 q/v 投影处加 delta，相当于轻量调整注意力行为，而不是全量重训 Qwen。

save_pretrained

作用：Hugging Face/PEFT 常用保存接口。对 PEFT model 调用时，通常保存 adapter 配置和权重，不保存完整 base model。

例子：model.save_pretrained(adapter_dir) 会保存 adapter_config.json 和 adapter 权重文件。

PeftModel.from_pretrained

作用：把已经保存的 PEFT adapter 加载回 base model，推理部署常用“先加载 base，再加载 adapter”。

例子：先加载 fresh Qwen，再用 PeftModel.from_pretrained(base_model, adapter_dir) 挂回本课训练出的 LoRA。

trainable ratio

作用：可训练参数占总参数的比例，用来衡量参数高效微调到底省了多少训练量。

例子：本课只训练 540,672 个参数，占约 494.6M 总参数的 0.1093%。

PEFT METHOD MAP

PEFT 还有哪些具体方法

PEFT 的共同点是少训练参数。你当前应该先吃透 LoRA，再把其他方法当作扩展地图。

LoRA / AdaLoRA

LoRA 在目标线性层加 A/B 低秩矩阵；AdaLoRA 会动态分配 rank，把参数预算给更重要的层。

IA3

不加低秩矩阵，而是学习少量缩放向量，调节 attention 或 FFN 中的激活流。

Prompt / Prefix / P-Tuning

不主要改模型层，而是训练 soft prompt、每层 prefix key/value，或 prompt encoder。

LoHa / LoKr

LoRA 的分解变体，用 Hadamard 或 Kronecker 结构表达 adapter 增量。

OFT / BOFT

通过正交变换方式调整权重表示，属于更进阶的参数高效方法。

X-LoRA / LayerNorm Tuning

X-LoRA 用门控组合多个 LoRA adapter；LayerNorm Tuning 只训练极少数 norm 参数。
