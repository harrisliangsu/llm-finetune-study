MODEL CHANGE FLOW

LoRA 让模型发生了什么变化

训练前只有 frozen base。注入 LoRA 后，原始 Qwen 权重仍然不动，训练只把梯度写入 q_proj / v_proj 旁边新增的低秩矩阵。

Before
原始 Qwen Base
完整模型参数用于前向计算，但本课不会更新这些参数。
Embedding / MLP / Normfrozen
Attention q_projfrozen
Attention v_projfrozen
→
Inject
挂载 LoRA Adapter
PEFT 在目标线性层旁边增加 A/B 两个低秩矩阵。
q_proj: W + B(Ax) * scaleA/B
v_proj: W + B(Ax) * scaleA/B
其他原始层frozen
→
After
保存行为补丁
训练结束只保存 adapter。推理时再把 adapter 加回 fresh base。
adapter_model.safetensorssaved
adapter_config.jsonsaved
Base model 权重not saved
forward: output = frozen_base(x) + lora_B(lora_A(x)) * scaling backward: loss -> gradients -> LoRA A/B only base gradients are blocked because base weights are frozen
Base Model 路径
输入 token 会经过 Qwen 原始层，负责保留原模型知识和语言能力。参数参与 forward，但不参与 optimizer 更新。
LoRA Adapter 路径
同一批 SFT 数据产生的 loss 只更新 LoRA A/B。它学到的是“如何调整 q_proj/v_proj 的输出”，所以能作为小文件单独保存。
